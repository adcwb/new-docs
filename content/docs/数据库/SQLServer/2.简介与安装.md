这篇介绍 SQL Server 的版本分类、安装的软硬件要求、服务的启停方式，以及连接数据库实例的几种方式。

## 版本分类

SQL Server 针对不同应用场景提供多个版本：

| 版本 | 说明 |
| --- | --- |
| Express | 适用于无连接的客户端或独立应用程序，免费 |
| Workgroup | 适用于工作组或分支机构操作的数据库 |
| Standard | 部门级应用程序的数据库服务器 |
| Enterprise | 高度可伸缩、高度可用的企业级数据库 |
| Developer | 功能等同 Enterprise，但仅授予开发和测试用许可 |
| Web | 供托管公司提供低成本、高伸缩的托管服务，按月收取低廉许可费 |
| Mobile | 用于智能手持设备的精简数据库 |

## 安装要求

**硬件要求**：处理器、内存和磁盘空间需满足对应版本的最低要求（生产环境建议在最低要求基础上留出余量）。

**软件要求**：

- 安装前需具备 **.NET Framework 3.5** 组件（部分组件依赖该框架）。
- 受支持的 Windows 操作系统版本。
- 安装管理工具 **SQL Server Management Studio（SSMS）** 用于图形化管理。

{{< callout type="info" >}}
安装时会区分**实例**：默认实例直接用主机名/IP 访问，命名实例需用 `主机名\实例名` 访问。
{{< /callout >}}

## 服务管理

SQL Server 以 Windows 服务的形式运行，启动或停止服务有三种途径：

1. 【控制面板】→【管理工具】→【服务】中启动或停止。
2. 【SQL Server 配置管理器】中启动或停止。
3. 【SQL Server Management Studio】中启动或停止。

## 连接方式

连接到 SQL Server 实例时，"服务器名称"可以填写：

- **本地**：`(local)` 或者 `.`（一个英文句点）。
- **远端**：IP 地址、FQDN（完全限定域名）或主机名都可以。
