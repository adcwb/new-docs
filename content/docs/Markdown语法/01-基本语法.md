本文涵盖 Markdown 最常用的基本语法，适合日常写作快速参考。所有语法在标准 Markdown（CommonMark / GitHub Flavored Markdown）中均可使用。

## 标题

Markdown 支持两种标题写法：**ATX 风格**（推荐）和 **Setext 风格**。

### ATX 风格

使用 `#` 号开头，数量对应标题级别（1~6）：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### Setext 风格

一级标题和二级标题可以用底线形式（不常用）：

```markdown
一级标题
=======

二级标题
-------
```

## 段落与换行

- **分段**：两段之间插入空行。
- **换行**：在行尾加两个空格后回车，或使用 `<br>` 标签。
- 段落首行**不需要缩进**，靠空行区分段落即可。

```markdown
这是第一段。

这是第二段（中间有空行）。
第一行（行尾两个空格）  
第二行（紧接在上面那行之后）。
```

## 强调

| 语法 | 效果 |
| :--- | :--- |
| `**粗体**` 或 `__粗体__` | **粗体** |
| `*斜体*` 或 `_斜体_` | *斜体* |
| `***粗斜体***` | ***粗斜体*** |
| `~~删除线~~` | ~~删除线~~ |
| `` `行内代码` `` | `行内代码` |

使用建议：中文写作优先用 `**粗体**` 和 `*斜体*`（星号），避免下划线在中文语境中与专有名词混淆。

## 列表

### 无序列表

使用 `-`、`*` 或 `+` 作为标记，推荐统一使用 `-`：

```markdown
- 项目一
- 项目二
  - 嵌套项目（缩进 2 个空格）
  - 嵌套项目二
- 项目三
```

### 有序列表

使用数字 + `.` 开头：

```markdown
1. 第一步
2. 第二步
   1. 子步骤（缩进 3 个空格）
   2. 子步骤二
3. 第三步
```

### 任务列表

```markdown
- [x] 已完成任务
- [ ] 待办任务
- [ ] 另一项待办
```

## 链接与图片

### 行内链接

```markdown
[链接文字](https://example.com)
[带标题的链接](https://example.com "鼠标悬停提示")
```

### 引用式链接

适合多次引用同一 URL：

```markdown
[GitHub][gh]
[GitHub 首页][gh]

[gh]: https://github.com "GitHub"
```

### 自动链接

用 `<>` 包裹 URL 或邮箱：

```markdown
<https://example.com>
<user@example.com>
```

### 图片

```markdown
![替代文字](图片路径.png)
![替代文字](图片路径.png "图片标题")
```

站内图片推荐使用相对路径，将图片与文档放在同一目录。

## 代码

### 行内代码

用单个反引号包裹：

```markdown
使用 `fmt.Println()` 输出一行文本。
```

如果代码内含有反引号，用双反引号包裹：

```markdown
`` 包含 ` 反引号的代码 ``
```

### 围栏代码块

用三个反引号（`` ``` ``）包裹，并标注语言：

````markdown
```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```text
````

常用语言标签：`bash`、`python`、`go`、`javascript`、`sql`、`yaml`、`json`、`html`、`css` 等。

## 表格

使用 `|` 分隔列，`-` 定义表头分隔行：

```markdown
| 姓名 | 年龄 | 城市 |
| :--- | ---: | :---: |
| 张三 | 28 | 北京 |
| 李四 | 32 | 上海 |
```

对齐方式：`:---` 左对齐，`---:` 右对齐，`:---:` 居中。

## 引用

使用 `>` 作为引用前缀：

```markdown
> 这是一段引用。
>
> 引用内可以分段。
>
> > 引用可以嵌套。
```

## 分隔线

三个或更多的 `---`、`***`、`___`：

```markdown
---

***
```

> 注意：`---` 前后应有空行，否则可能被解析为 Setext 标题。

## 转义字符

使用反斜线 `\` 转义特殊符号：

```markdown
\* 不是斜体 \*
\# 不是标题
\> 不是引用
```

## 原始 HTML

Markdown 允许直接嵌入 HTML 标签：

```markdown
<details>
<summary>点击展开</summary>

这里的内容默认折叠。

</details>

<kbd>Ctrl</kbd> + <kbd>C</kbd>
```

## 脚注

```markdown
这是一段带脚注的文字[^1]。

[^1]: 这是脚注内容。
```

> 脚注在 GitHub Flavored Markdown 和部分静态站点生成器中支持，但并非所有 Markdown 引擎都支持。
