这篇讲三个保障网络安全与可运维的技术：用 **ACL** 过滤流量、用 **AAA** 实现认证授权计费，以及华为的统一运维管理系统 **eSight**。

## ACL 访问控制列表

**ACL（Access Control List，访问控制列表）**是路由器和交换机接口的指令列表，用来控制端口进出的数据包。它适用于所有被路由协议，如 IP、IPX、AppleTalk 等。

为保证内网安全，需要通过安全策略保障非授权用户只能访问特定网络资源。简而言之，**ACL 可以过滤网络中的流量，是控制访问的一种网络技术手段**。

ACL 的作用：

- **限制网络流量、提高网络性能**：例如根据协议指定数据包的优先级。
- **提供通信流量的控制手段**：例如限定或简化路由更新信息的长度，从而限制某网段的通信流量。
- **提供基本的网络安全访问手段**：例如允许主机 A 访问人力资源网络，而拒绝主机 B。
- **决定哪种流量被转发或阻塞**：例如允许 E-mail 流量被路由，拒绝所有 Telnet 流量。

典型应用：某部门只允许使用 WWW 功能，可用 ACL 实现；为保密某部门，不允许其访问外网、也不允许外网访问它，同样用 ACL 实现。

ACL 既可在路由器上配置，也可在具备 ACL 功能的业务软件上配置，是物联网等系统中保障安全性的重要技术。

## AAA

**AAA** 是 **Authentication（认证）、Authorization（授权）、Accounting（计费）**的简称，提供这三种安全功能：验证用户账户是否合法、授权可访问的服务、记录用户使用网络资源的情况。

华为设备支持基于两种协议实现 AAA：

| 协议 | 全称 | 特点 |
| --- | --- | --- |
| RADIUS | Remote Authentication Dial-In User Service | 基于 UDP；认证与授权绑定；只加密密码 |
| HWTACACS | Huawei Terminal Access Controller Access Control System | 华为对 TACACS 的扩展，基于 TCP；认证授权独立；可加密整个报文 |

**RADIUS 与 HWTACACS 的区别**：

1. RADIUS 基于 UDP，HWTACACS 基于 TCP。
2. RADIUS 的认证和授权绑定，HWTACACS 的认证和授权独立。
3. RADIUS 只加密密码，HWTACACS 可加密整个报文。

应用场景：企业总部对服务器资源访问进行控制，只有通过认证的用户才能访问特定资源，并记录使用情况。**NAS**（网络接入服务器）负责集中收集和管理用户访问请求；**AAA 服务器**（RADIUS 或 HWTACACS）负责制定认证、授权和计费方案。

### 认证

AAA 支持三种认证方式：

- **不认证**：完全信任用户，不检查身份。安全性差，很少采用。
- **本地认证**：用户信息配置在 NAS 上。处理快、成本低，但受设备硬件存储限制。
- **远端认证**：用户信息配置在认证服务器上，通过 RADIUS 或 HWTACACS 远端认证。

如果一个方案采用多种认证方式，按配置顺序生效。例如先配远端、后配本地，则远端无响应时转入本地认证。

### 授权

AAA 授权赋予用户访问特定网络或设备的权限。支持的授权方式：

- **不授权**：不对用户授权处理。
- **本地授权**：根据 NAS 上本地账号的属性授权。
- **远端授权**：HWTACACS 授权用 TACACS 服务器；RADIUS 授权对通过 RADIUS 认证的用户授权（RADIUS 认证授权绑定，不能单独用 RADIUS 授权）。

多种授权方式按配置顺序生效，不授权方式最后生效。

### 计费

计费用于监控授权用户的网络行为和资源使用情况。支持两种方式：

- **不计费**：提供免费上网，不产生活动日志。
- **远端计费**：通过 RADIUS 或 HWTACACS 服务器计费，服务器储存各用户的访问活动日志。

> 目前华为 ARG3 系列路由器只支持配置**认证和授权**，计费方案需配置在 HWTACACS 或 RADIUS 服务器上。

### AAA 域

AAA 通过**域**来管理用户，不同域可关联不同的认证、授权、计费方案。每个用户都属于某一个域，由用户名中域名分隔符 `@` 后的字符串决定。

- 例如 `user@huawei` 属于 `huawei` 域；用户名不带 `@` 则属于系统缺省域 `default`。
- ARG3 系列支持两个缺省域：`default`（普通用户）和 `default_admin`（管理用户），可修改但不能删除。默认最多支持 32 个域（含两个缺省域）。

### 配置示例

```text
[RTA]aaa
[RTA-aaa]authentication-scheme auth1
[RTA-aaa-authen-auth1]authentication-mode local
[RTA-aaa-authen-auth1]quit
[RTA-aaa]authorization-scheme auth2
[RTA-aaa-author-auth2]authorization-mode local
[RTA-aaa-author-auth2]quit
[RTA-aaa]domain huawei
[RTA-aaa-domain-huawei]authentication-scheme auth1
[RTA-aaa-domain-huawei]authorization-scheme auth2
[RTA-aaa-domain-huawei]quit
[RTA-aaa]local-user huawei@huawei password cipher huawei
[RTA-aaa]local-user huawei@huawei service-type telnet
[RTA-aaa]local-user huawei@huawei privilege level 0
[RTA]user-interface vty 0 4
[RTA-ui-vty0-4]authentication-mode aaa
```

常用命令：

| 命令 | 作用 |
| --- | --- |
| `authentication-scheme <name>` | 配置域的认证方案（缺省 `default`） |
| `authentication-mode {hwtacacs \| radius \| local}` | 配置认证方式（缺省本地认证） |
| `authorization-scheme <name>` | 配置域的授权方案 |
| `authorization-mode {hwtacacs \| if-authenticated \| local \| none}` | 配置授权方式（缺省本地授权） |
| `domain <name>` | 创建并进入 AAA 域视图 |
| `local-user <name> password cipher <pwd>` | 创建本地用户并配置密码 |
| `local-user <name> privilege level <level>` | 指定本地用户优先级 |
| `display domain [name <name>]` | 查看域配置（`Active` 表示激活状态） |

## eSight 网络管理

随着企业网络结构日益复杂，且常包含多厂商设备，若每种设备都用各自的网管平台，会给管理带来很大不便。

**eSight** 是华为面向企业市场推出的新一代网络运维管理系统，面向企业有线/无线园区、企业分支网络、数据中心网络，支持**多厂商、多种类设备**的管理，能实现对企业资源、业务、用户的统一管理及智能联动，极大提升网络管理效率。

主要功能特性：

- **基础网络管理、网元管理、业务管理、系统管理**等全面功能。
- **配置文件管理**：对设备配置文件进行导入、备份、恢复等操作。
- **WLAN 管理**：为企业 WLAN 网络提供一体化管理方案，实现监控、配置管理、故障诊断等。
- **网络流量分析（NTA，Network Traffic Analyzer）**：作为管理组件对网络流量进行采集与分析。
