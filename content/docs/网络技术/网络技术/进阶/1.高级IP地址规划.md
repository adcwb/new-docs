IP 地址的合理规划是网络设计中的重要一环，直接影响路由协议算法的效率、路由收敛速度，以及网络的稳定性、可扩展性和整体性能。这篇讲 IP 规划的重要性、用 VLSM 进行子网规划的方法，以及路由聚合与 CIDR。地址分类、特殊地址等基础参见[网络层](../7.网络层.md)。

## 为什么要规划 IP 地址

现代网络规模越来越大，普遍采用**分层结构**：

- **核心层**：完成数据包交换，实现高速流量转发，要求容量大、转发快、高稳定。
- **汇聚层**：隔离拓扑结构变化、控制路由表大小、控制网络收敛，并实现丰富业务特性。
- **接入层**：把终端用户接入网络，需要大量端口和强大接入能力。

这种分层结构要求在分配 IP 地址时必须**详细规划**。规划的好坏会直接影响：

- 路由协议算法的效率
- 网络的性能
- 网络的扩展能力
- 网络的管理

{{< callout type="info" >}}
**规划的价值举例**：如果区域内地址连续且合理规划，核心层可以把多个网段**聚合**成一条路由发布到其他区域——每台路由器只需维护"本区域明细 + 其他区域聚合"少量路由。若不做规划无法聚合，核心路由器要向其他区域发布本区域**所有**网段路由，路由条目数可能翻倍。
{{< /callout >}}

## 子网掩码与地址计算

**子网掩码（subnet mask）**用来区分 IP 地址的网络部分和主机部分：

- 使用与 IP 地址一样的点分十进制格式。
- 网络部分和子网部分全为 **1**，主机部分全为 **0**。
- 缺省掩码：A 类 `255.0.0.0`、B 类 `255.255.0.0`、C 类 `255.255.255.0`。

两种表示法等价：`255.255.255.240` 与 `/28` 都表示前 28 bit 为网络号。

### 网络地址的计算

已知 IP 和掩码，把二者按位**"与"运算**（`1&1=1`，其余为 0）即得网络地址。例如 `192.168.1.7/28`：

```text
  11000000.10101000.00000001.00000111   (192.168.1.7)
& 11111111.11111111.11111111.11110000   (255.255.255.240)
= 11000000.10101000.00000001.00000000   (192.168.1.0  ← 网络地址)
```

### 主机数与子网数

设主机位为 $n$、借用的子网位为 $m$：

- 每个子网的**可用主机数** = $2^n - 2$（减去全 0 网络地址和全 1 广播地址）。
- 划分出的**子网数** = $2^m$。

例：`192.168.1.100/28`，`/28` 即 `255.255.255.240`，主机位 $n=4$ → 主机总数 $2^4=16$，可用主机数 $2^4-2=14$。

## VLSM 可变长子网掩码

把一个网络划分成多个子网时，各子网的主机数往往相差很大。如果所有子网都用**固定长度**掩码，会造成大量地址浪费。

**VLSM（Variable Length Subnet Masking，可变长子网掩码）**允许不同子网使用不同长度的掩码：

- 节点多的子网用**较短**掩码（可分配地址多）。
- 节点少的子网用**较长**掩码（可分配地址少）。

这样能节省大量地址，节省出来的地址可用于其他子网。

### 子网规划实例

**需求**：某公司用 C 类地址 `192.168.1.0` 规划。共 5 台路由器：1 台作为网关接入 ISP，其余 4 台各连一个办公点，每个办公点 20 台 PC。

**步骤 1：确定子网位与主机位。** 4 个办公点每个需 21 个地址（含路由器接口）。按 $2^n - 2 \ge 20$ 求得 $n=5$（主机位 5 位），则子网位 $= 8 - 5 = 3$ 位。

**步骤 2：列出子网位组合。** 主机部分的高 3 位为子网位，从 `000` 开始递增枚举所有组合。

**步骤 3：换算为点分十进制网段地址。** 选取其中连续的 4 个作为办公点网段：

| 办公点 | 网段 | 可用主机范围 |
| --- | --- | --- |
| 1 | 192.168.1.32/27 | .33 ~ .62 |
| 2 | 192.168.1.64/27 | .65 ~ .94 |
| 3 | 192.168.1.96/27 | .97 ~ .126 |
| 4 | 192.168.1.128/27 | .129 ~ .158 |

办公点路由器与网关路由器互联的网段（每段只需 2 个地址）可继续用更长掩码（如 `/30`）从剩余地址中切分，进一步节省地址。

## 路由聚合

**路由聚合**是将多条路由聚合成一条，可大大减少路由器中的路由条目数，减轻维护负担、提高网络利用率。

例：RTA 下接 4 个网段 `172.1.12.0/24`、`172.1.13.0/24`、`172.1.14.0/24`、`172.1.15.0/24`，可聚合为 `172.1.12.0/22`，向 RTB 通告时只发这一条。

## CIDR 无类域间路由

**CIDR（Classless Inter-Domain Routing，无类域间路由，RFC 1817）**突破了传统 IP 地址的分类边界，使用 VLSM 技术把路由表中的若干条路由汇聚为一条，减小路由表规模、提高路由器可扩展性。

支持 CIDR 的路由协议：RIPv2、OSPF、Integrated IS-IS、BGPv4。

例：某 ISP 分到 `198.168.0.0 ~ 198.168.255.0` 的一批 C 类网络，已分配 `198.168.1.0/24`、`198.168.2.0/24`、`198.168.3.0/24` 给用户。实施 CIDR 后，ISP 路由器只向 Internet 通告 `198.168.0.0/16` 这一条，大大减少路由条目。

{{< callout type="warning" >}}
CIDR 汇聚要求被汇聚网络地址的比特位必须一致。若该 ISP 又接了一个 `172.178.1.0/24` 网段，这些网段就无法汇聚。（极端情况下可汇聚成 `0.0.0.0/0` 发布。）
{{< /callout >}}

### 路由汇总与 CIDR 汇总的区别

- **路由汇总**仍有"类"的概念：汇总后的掩码长度必须 **大于或等于**主类网络的掩码长度。
- **CIDR** 是无类的：只要网络地址比特一致就能汇总，与主类边界无关。
