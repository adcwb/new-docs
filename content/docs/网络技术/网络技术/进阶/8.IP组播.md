IP 组播有效解决了点到多点应用的效率问题：组播源只发送一份数据，数据在网络节点间被复制、分发，且只发送给需要它的接收者。这篇讲组播的优势、组播 IP 地址与模型（ASM/SSM）、地址范围以及组播 MAC 地址。

## 组播的优势

相比单播逐个发送、广播全网泛洪，组播在点到多点场景下：

- **无重复流量**：源只发一份，沿途按需复制。
- **节省设备与带宽资源**。
- **安全性高**：只有加入组的接收者才收到。
- **有偿性有保障**：便于按组提供有偿服务。

## 组播 IP 地址

一个组播 IP 地址并不表示某台具体主机，而是**一组主机的集合**。主机声明加入某组播组，即表示自己要接收目的地址为该组播地址的数据。

| 范围 | 含义 |
| --- | --- |
| 224.0.0.0 ~ 224.0.0.255 | 为路由协议预留的永久组地址 |
| 224.0.1.0 ~ 231.255.255.255 / 233.0.0.0 ~ 238.255.255.255 | Any-Source 临时组播组地址 |
| 232.0.0.0 ~ 232.255.255.255 | Source-Specific 临时组播组地址 |
| 239.0.0.0 ~ 239.255.255.255 | 本地管理的 Any-Source 临时组播组地址 |

## 组播模型

IP 组播常见模型分为 ASM 和 SSM：

| 模型 | 全称 | 特点 |
| --- | --- | --- |
| ASM | Any-Source Multicast（任意源组播） | 任意发送者都可成为组播源；接收者加入组后能收到发往该组的所有信息，无法预先知道源的位置 |
| SSM | Source-Specific Multicast（指定源组播） | 接收者已通过其他手段预先知道组播源的具体位置，可在客户端指定信源 |

二者根本区别在于：**接收者是否预先知道组播源的位置**。SSM 和 ASM 使用不同的组播地址范围；SSM 直接在接收者和组播源之间建立组播转发树。

## 组播 MAC 地址

IANA 规定，IPv4 组播 MAC 地址的高 24 位为 `0x01005e`，第 25 位固定为 0。其中 `0x` 只是十六进制标识，本身无意义。
