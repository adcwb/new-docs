执行路由策略或策略路由前，通常要先把特定的路由信息或数据包"过滤"出来。这篇讲常用的过滤/匹配工具（ACL、前缀列表、as-path-filter、community-filter、route-policy）、它们各自的适用对象，以及策略路由与路由策略的区别。最后附各路由协议的默认优先级。

## 选对工具：先看匹配对象

不同工具匹配的对象不同，要先弄清是匹配**路由**还是**数据包**：

| 工具 | 匹配对象 | 能否过滤数据包 | 适用范围 |
| --- | --- | --- | --- |
| ACL | 数据包地址 / 路由信息 | 可以 | 通用 |
| 前缀列表（ip-prefix） | 路由的目的地址 | 不可以 | 仅路由信息 |
| as-path-filter | BGP 路由的 AS_PATH | 不可以 | 仅 BGP |
| community-filter | BGP 路由的团体属性 | 不可以 | 仅 BGP |
| route-policy | 路由 / 数据包（可调用其他工具） | 可以 | 通用，且能修改属性 |

## ACL 访问控制列表

ACL 既可匹配数据包，也可匹配路由信息：

- **基本 ACL**：只能匹配源 IP 地址。
- **高级 ACL**：可匹配源/目的 IP、源/目的端口号、协议号等（两个以上条件）。
- **基于接口的 ACL**：匹配接口。

### 匹配顺序

ACL 有两种匹配顺序：

- **配置顺序**：按用户配置规则的先后匹配。
- **自动排序**：使用"深度优先"原则——把指定范围最小的语句排在最前。

"深度优先"通过比较地址通配符实现：**通配符越小，指定主机范围越小，排越前**。例如 `129.102.1.1 0.0.0.0` 指定一台主机，`129.102.1.1 0.0.0.255` 指定一个网段，前者排前面。

具体标准：

- 基本 ACL：比较源地址通配符，相同则按配置顺序。
- 基于接口的 ACL：配 `any` 的规则排后面，其余按配置顺序。
- 高级 ACL：先比源地址通配符，再比目的地址通配符，再比端口号范围（范围小者靠前），都相同则按配置顺序。

## 前缀列表（ip-prefix）

前缀列表用来过滤 IP 前缀，能**同时匹配前缀号和前缀长度**，性能比 ACL 高，但只能过滤路由信息、不能过滤数据包。

```text
ip ip-prefix test index 10 permit 10.0.0.0 16 greater-equal 24 less-equal 28
```

含义：前缀号必须为 `10.0`，且 `24 <= 前缀长度 <= 28`。满足条件的如 `10.0.1.0/24`、`10.0.2.0/25`、`10.0.2.192/26`。

## BGP 专用过滤器

- **as-path-filter**：匹配 BGP 路由信息中的 AS_PATH 属性，仅用于过滤 BGP 路由。
- **community-filter**：匹配 BGP 路由信息中的团体属性，仅用于过滤 BGP 路由。

## route-policy 路由策略

route-policy 是强大的过滤工具，同时也是策略工具，由一系列 `if-match` 子句和 `apply` 子句构成。

- **if-match 子句**：匹配条件，可引用 acl、ip-prefix、as-path-filter、community-filter、interface、cost、tag、route-type 等。
- **apply 子句**：修改路由的属性或数据包的转发行为。

结构与逻辑关系：

- 一个 route-policy 下可有多个**节点**，用 `seq-number` 标识，不同节点之间是**"或"**的关系。
- 每个节点下可有多个子句，多个 `if-match` 子句之间是**"与"**的关系。
- **允许（permit）模式**：路由满足该节点所有 if-match 则通过过滤并执行 apply；不满足则尝试匹配下一个节点。
- **拒绝（deny）模式**：路由满足该节点所有 if-match 则被拒绝，且不再测试后续节点。

常用比较关键字：`eq`（等于）、`gt`（大于）、`lt`（小于）。网络位又称**前缀**。

## 策略路由 vs 路由策略

二者常被混淆，核心区别在于"控制谁"：

| | 策略路由（Policy-Based Routing） | 路由策略（Routing Policy） |
| --- | --- | --- |
| 控制对象 | **报文的转发** | **路由信息**的引入、发布、接收 |
| 行为 | 可不按路由表转发报文，随心所欲把报文转发出去 | 控制哪些路由引入/发布/接收，哪些丢弃 |

- **策略路由**：一般报文转发要查转发表，配上策略路由后可不查转发表。
- **路由策略**：主要针对某种路由协议——控制引入（哪些路由引入协议）、发布（哪些发布出去）、接收（哪些接收、哪些丢弃）。

## 路由协议默认优先级

不同协议来源的路由用优先级（preference）判断可信度，**值越小越优先**。`preference` 命令可修改优先级。

| 路由来源 | 默认优先级 |
| --- | --- |
| 直连（Direct） | 0 |
| 静态（Static） | 60 |
| OSPF | 10 |
| IS-IS | 15 |
| RIP | 100 |
| 内部 BGP（IBGP） | 130 |
| OSPF 外部（External） | 150 |
| 外部 BGP（EBGP） | 170 |
