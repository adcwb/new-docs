这篇讲两个与网络质量相关的技术：**QoS**（服务质量，为关键业务提供可预期的服务水平）和 **NQA**（网络质量分析，通过模拟报文测试为优化网络提供数据）。

## QoS 度量指标

**QoS（Quality of Service，服务质量）**为关键业务提供服务质量保证，使其获得可预期的服务水平。它用以下参数来度量：

| 指标 | 说明 |
| --- | --- |
| 带宽 / 吞吐量 | 单位时间内可传输的数据量 |
| 时延 | 报文从源到目的的传输时间 |
| 时延变化（抖动 Jitter） | 时延的波动程度 |
| 丢包率 | 传输过程中丢失报文的比例 |

## NQA 网络质量分析

**NQA（Network Quality Analysis，网络质量分析）**所有操作基于模拟——把测试流量模拟成各类报文进行测试，为优化网络提供数据。它不是一个协议，而是一种架构技术，可在应用层、网络层（端口、链路）、传输层进行测试。运营商（SP）使用 NQA 较多。

组网模型：**C/S 架构（client/server）**。

常见测试类型：

- **应用层测试例**：DHCP、SNMP（简单网络管理协议）、FTP（模拟上传下载）、DNS（解析请求测试）、HTTP。
- **Jitter（抖动）测试**：测量时延变化，常用指标 **RTT（AVG）**——平均往返时间。
- **ICMP 测试**。
- **NQA 联动技术**：测试结果可与路由等联动（如主备切换）。

> 路由器、交换机普遍具备 NQA 特性，但通常还有专门的测试软件/客户端。相关指标 SLA（服务等级协议）常与 NQA 配合使用。

常用命令：

```text
start now        # 立即开始测试
display nqa      # 查看 NQA 结果
```
