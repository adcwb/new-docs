**BGP（Border Gateway Protocol，边界网关协议）**是在自治系统之间传递路由信息的外部网关协议，是当今 Internet 域间路由的事实标准。这篇讲 BGP 的特点、报文类型、邻居状态机、路径属性、选路过程以及路由反射。

## 基础概念

- **自治系统（AS）**：由同一技术管理机构管理、使用统一选路策略的一组路由器集合。
- **IGP（内部网关协议）**：AS 内部路由协议，如 RIP、OSPF、IS-IS，负责"计算"路径。
- **EGP（外部网关协议）**：AS 之间路由协议，即 BGP，负责"选路"。

AS 编号范围 1~65535：`1~64511` 为注册的因特网编号，`64512~65535` 为私有编号。

## BGP 特点

- 外部路由协议，用于在 AS 之间传递路由信息。
- 是一种**增强的距离矢量（路径矢量）**路由协议，从设计上避免环路。
- 使用 **TCP（端口 179）**作为承载协议，提高可靠性；**无需直连**即可建立邻居。
- 用**单播**发送消息，不需要组播地址。
- 无需周期性发送完整更新，只发送**增量路由**；周期性发送 KeepAlive 报文检测 TCP 连通性。
- 支持 CIDR，为路由附带丰富属性，路由过滤和策略丰富，易于扩展，路由条目无上限。

两种对等体关系：

| 类型 | 含义 | 优先级 |
| --- | --- | --- |
| EBGP | AS 与 AS 之间建立的 BGP 关系 | 170 |
| IBGP | AS 内部建立的 BGP 关系 | 130 |

> 运行 BGP 的路由器称 **BGP Speaker**，建立连接的双方互为**对等体（peer）**。BGP 邻居关系建立在 TCP 连接之上，TCP 的 IP 可达性可由 IGP 或静态路由提供。在一个 IBGP 内部一定要运行一个 IGP。

## 报文类型

| 报文 | 作用 |
| --- | --- |
| Open | 协商 BGP 参数（版本、AS 号等），建立邻居关系 |
| Keepalive | 周期性发送，维护连接、判断对等体可达性 |
| Update | 交换路由信息（撤销路由 + 可达路由及属性），最重要的报文 |
| Notification | 检测到错误时发送，随后关闭相关邻居关系 |
| Route-refresh | 通知对等体自己支持路由刷新能力；改变路由策略后请求对等体重发路由 |

前四种定义在 RFC 4271，Route-refresh 定义在 RFC 2918。

## 邻居状态机

BGP 有六种状态：

| 状态 | 说明 |
| --- | --- |
| Idle | 初始状态，拒绝任何进入的连接请求，等待启动事件 |
| Connect | 发起第一个 TCP 连接；成功转 OpenSent，失败转 Active |
| Active | 不断尝试建立 TCP 连接；成功转 OpenSent |
| OpenSent | TCP 已建立并发出 Open 报文，等待对端 Open 并检查正确性 |
| OpenConfirm | 等待 Keepalive，收到后转 Established |
| Established | 邻居关系建立，交换 Update 报文 |

除 Idle 外的任何状态出现 Error，状态机都会退回 Idle。

{{< callout type="info" >}}
对等体建立过程中通常可见三个状态：**Idle**（初始，拒绝连接）、**Active**（中间态，尝试 TCP 连接）、**Established**（已建立，可交换 Update/Route-refresh/Keepalive/Notification）。**双方都必须为 Established，邻居关系才成立。**
{{< /callout >}}

## BGP 路由表

| 表 | 说明 |
| --- | --- |
| IP 路由表（IP-RIB） | 全局路由信息库，含所有 IP 路由 |
| BGP 路由表（Loc-RIB） | 本地 BGP Speaker 选择的路由 |
| 邻居表 | 对等体邻居清单 |
| Adj-RIB-In | 对等体宣告给本地、未处理的路由 |
| Adj-RIB-Out | 本地宣告给指定对等体的路由 |

查看命令 `display bgp routing-table`，路由前的标记：`*` 有效、`>` 最优、`s` 被抑制、`i` 内部、`d` 死亡、`h` 历史；起源 `i(IGP) > e(EGP) > ?(未知)`。

## 通告原则

1. 连接一建立，BGP Speaker 把自己所有 BGP 路由通告给新对等体；多条路径只选最优给自己用，且只把自己使用的最优路由通告给对等体。
2. 从 EBGP 获得的路由会通告给所有 BGP 对等体（含 EBGP 和 IBGP）。
3. 从 IBGP 获得的路由**不会**通告给其他 IBGP 邻居（防环，需用全互联或路由反射解决）。
4. 从 IBGP 获得的路由是否通告给 EBGP 对等体，取决于 IGP 与 BGP 的同步情况（华为 VRP 缺省非同步且不可改）。
5. **下一跳**：向 EBGP 通告时下一跳设为自身 IP；向 IBGP 通告时不改变下一跳，需保证 IBGP 下一跳可达。可用 `peer x.x.x.x next-hop-local` 修改为本地。

## 路径属性

BGP 属性分四类：公认必遵、公认任意、可选过渡、可选非过渡。

| 属性 | 类别 | 说明 |
| --- | --- | --- |
| Origin（起源） | 公认必遵 | 标记路由如何成为 BGP 路由：i(IGP) > e(EGP) > ?(Incomplete) |
| AS_PATH | 公认必遵 | 路由经过的 AS 序列，防环、用于过滤选择；**路径越短越优先**，`0.0.0.0` 代表本地产生 |
| Next_hop | 公认必遵 | 下一跳边界路由器 IP |
| MED | 可选非过渡 | 多入口时帮外部 AS 选最优入口，相当于 cost，**越小越优**，影响 EBGP 邻居 |
| Local-Preference | 公认任意 | AS 内优选路由，**越大越优**，默认 100，只在本地有意义，影响 IBGP 邻居 |
| Community（团体） | 可选过渡 | 标识一组具有相同特征的路由，与所在子网/AS 无关 |

修改命令：`default med <值>`、`default local-preference <值>`。

## 选路过程

BGP 按以下顺序选最优路由（前面命中即决定）：

1. 下一跳不可达，忽略此路由。
2. **Preferred-Value（偏好值）**高者优先。
3. **Local-Preference** 最高者优先。
4. 聚合路由优于非聚合路由。
5. 手动聚合优于自动聚合。
6. `network` 引入的路由优于 `import-route` 引入的路由。
7. **AS_PATH** 最短者优先。
8. 比较 **Origin**：IGP > EGP > Incomplete。
9. **MED** 较小者优先。
10. **EBGP（170）** 优于 **IBGP（130）**。
11. 到 BGP 下一跳的 **IGP 度量**最低者优先。

以上全部相同则为等价路由，可负载分担（此时 AS_PATH 必须一致，且后续 3 条原则失效）：

12. **Cluster-List** 短者优先。
13. **Originator_ID**（无则用 Router ID）较小者优先。
14. 对等体 **IP 地址**最小者优先。

## 路由黑洞与路由反射

IBGP 水平分割（从 IBGP 学到的路由不再通告给其他 IBGP 邻居）会导致部分路由器学不到路由，形成**路由黑洞**。解决方案：

- **IBGP 全互联（Full Mesh）**：所有 IBGP 对等体两两建立邻居，扩展性差。
- **路由反射（Route Reflection）**：设置**路由反射器（RR）**，将 IBGP 对等体分为**客户端**和**非客户端**。
- **联盟（Confederation）**：把一个 AS 划分为多个子 AS。

路由反射器配置：

```text
group <组名> internal              # 创建 IBGP 组（external 为 EBGP）
peer 10.0.12.2 group <组名>        # 邻居加入组
peer <组名> reflect-client         # 指定该组为客户端
```

RR 通告原则：

- 从**非客户机 IBGP 对等体**学到的路由，发布给该 RR 的所有客户机。
- 从**客户机**学到的路由，发布给该 RR 的所有非客户机和客户机（发起该路由的客户机除外）。
- 从 **EBGP 对等体**学到的路由，发布给所有非客户机和客户机。
