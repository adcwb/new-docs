这篇深入二层交换技术：VLAN 的划分方式与接口类型、VLAN 扩展技术（MUX VLAN、Super VLAN、QinQ），以及生成树家族 STP/RSTP/MSTP。VLAN 与 STP 的入门概念见[交换技术](../6.交换技术.md)。

## VLAN 划分方式

VLAN 技术把用户划分成多个逻辑网络（group），组内可通信、组间默认不通信，二层的单播/组播/广播报文只在组内转发，并能方便地增删组成员。其本质是**提供一种管理手段，控制终端之间的互通**。

> VLAN 1 是默认 VLAN，又称**本征 VLAN（Native VLAN）**。

五种划分方式：

| 方式 | 依据 | 说明 |
| --- | --- | --- |
| 基于端口 | 交换机端口 | 最简洁、最广泛使用，可把多个端口划入一个 VLAN |
| 基于 MAC | 报文源 MAC | 根据终端 MAC 确定报文在哪个 VLAN 转发 |
| 基于 IP 子网 | 报文源 IP 及掩码 | 如 1.1.1.0/24→VLAN10，1.1.2.0/24→VLAN20，对 untagged 报文按源 IP 打标签 |
| 基于协议 | 协议族及封装格式 | 如 IP/IPX/AppleTalk，Ethernet II/802.3 等，对 untagged 报文按协议类型打标签 |
| 基于策略 | IP+MAC（+Port）组合 | 对匹配策略的 untagged 报文进行 VLAN 划分 |

## VLAN 接口类型

| 类型 | 说明 |
| --- | --- |
| Access | 只允许一个 VLAN（PVID）通过，去标签 |
| Trunk | 允许多个 VLAN 通过；去 PVID 标签，其他允许的 VLAN 带标签通过 |
| Hybrid | 混合接口，可灵活地对不同 VLAN 去标签或加标签 |

## VLAN 扩展技术

### MUX VLAN

**MUX VLAN** 是一种二层流量隔离机制。典型场景：企业网中客户之间、客户与员工之间不可互访，员工之间可互访，员工与客户都可访问服务器。

结构：

- **Principal VLAN（主 VLAN）**
- **Subordinate VLAN（从 VLAN）**，又分：
  - **Group VLAN（互通型从 VLAN）**
  - **Separate VLAN（隔离型从 VLAN）**

通信权限：

1. 主 VLAN 端口可以和**所有** VLAN 通信。
2. 互通型从 VLAN 可以和**本 VLAN 成员**通信，以及和**主 VLAN** 通信。
3. 隔离型从 VLAN **只能和主 VLAN** 通信，本 VLAN 成员之间也不能通信。

配置注意：所有主机必须在同一子网；端口必须为 access 模式加入 VLAN；不能与 VLANIF 接口、VLAN Mapping、VLAN Stacking、Super VLAN、Sub-VLAN 配置共用。

### Super VLAN 与其他

- **Super VLAN**：配合 ARP 代理实现多个 Sub-VLAN 共用一个网段。ARP 代理分为路由式 ARP Proxy、VLAN 内 ARP Proxy、VLAN 间 ARP Proxy。
- **VLAN Mapping**：VLAN 的映射。
- **端口隔离 / 端口安全**：通过匹配 MAC 地址学习的数量来执行相应动作。

### QinQ

**QinQ** 基于 802.1Q 封装，在用户私网 VLAN tag 之外再封装一层公网 VLAN tag，公网中只根据外层公网 VLAN Tag 传播，为用户提供一种简单的二层 VPN 隧道。

优点：

- 解决日益紧缺的公网 VLAN ID 资源问题（内外层共可表示更多 VLAN）。
- 用户可规划自己的私网 VLAN ID，网络独立性高。
- 提供一种较简单的二层 VPN 解决方案。

分类：

- **基于端口的 QinQ**（基本 QinQ）。
- **灵活 QinQ**（VLAN Stacking）：基于流，可基于 ACL 等条件灵活封装。

优先级级别范围 0~7。

## 生成树家族

环路问题的演进逻辑：**单点故障 → 加冗余 → 冗余带来环路 → 用生成树消除环路**。

### STP

**桥 ID（Bridge ID）** = 优先级 + MAC。优先级越小越优先，**默认 32768**；修改时必须是 4096 的倍数。

**BPDU（Bridge Protocol Data Unit）**分为：配置 BPDU、TCN BPDU、TCA。

端口角色：

- **Root（根桥）**
- **RP（根端口）**：依据交换机选举，每台交换机有且只有一个。
- **DP（指定端口）**：依据网段选举，离根桥最近的端口。
- **NDP（预备端口）**：阻塞备用。

选举总结：

- 每台交换机有且只有一个根端口。
- 根桥上的所有端口都是指定端口。
- 根端口对应的对端一定是指定端口。

STP 缺陷：收敛速度慢、效率低下、端口角色多。（交换机 MAC 地址表老化时间 300s。）

### RSTP

RSTP（快速生成树）针对 STP 收敛慢的问题改进，简化端口状态和角色，实现秒级收敛。

### MSTP

**MSTP（多生成树）**用于解决**启用 VLAN 的交换网络**中的环路问题——一个生成树称为一个**实例（Instance）**，可把多个 VLAN 映射到不同实例分别计算，实现负载分担。

要点：

- 通过 MST 区域、MSTP 配置表组织。
- 通过配置端口优先级控制端口 ID，进而影响根端口、指定端口选举；端口优先级值越小，优先级越高。

MSTP 四种保护功能：BPDU 保护、根保护、环路保护、防 TC-BPDU 攻击保护。

> 相关标准：802.1d（STP）、802.1w（RSTP）、802.1s（MSTP）、802.1q（VLAN）、802.1p（优先级）、802.1t。
