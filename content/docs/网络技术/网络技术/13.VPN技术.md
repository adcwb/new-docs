传统 TCP/IP 协议缺乏有效的安全认证和保密机制。**IPSec** 作为开放标准的安全框架，可保证 IP 数据报在网络上传输的机密性、完整性和防重放。这篇讲 IPSec 的体系结构、安全联盟 SA、封装模式，以及一个手工建立 SA 的完整配置示例。

## IPSec 提供的安全能力

IPSec（Internet Protocol Security）是 IETF 定义的一个协议组，通信双方在 IP 层通过加密、完整性校验、数据源认证等方式保护 IP 报文。典型应用场景是企业远程分支机构通过 IPSec VPN 建立安全隧道接入企业总部网络。

它提供三种核心安全能力：

| 能力 | 含义 |
| --- | --- |
| 机密性（Confidentiality） | 对用户数据加密，以密文形式传送 |
| 完整性（Data integrity） | 对接收数据认证，判定报文是否被篡改 |
| 防重放（Anti-replay） | 拒绝旧的或重复的数据包，防止重放攻击 |

## IPSec 体系结构

IPSec 不是单一协议，主要由三部分组成：

| 组件 | 全称 | 作用 |
| --- | --- | --- |
| AH | Authentication Header | 数据源验证、数据完整性校验、防重放；**不加密**数据 |
| ESP | Encapsulating Security Payload | 除 AH 的功能外（完整性校验不含 IP 头），还提供**加密** |
| IKE | Internet Key Exchange | 自动协商 AH/ESP 使用的密码算法，建立和维护 SA |

### 安全联盟 SA

**SA（Security Association，安全联盟）**定义了对等体间使用的数据封装模式、认证和加密算法、密钥等参数。

- SA 是**单向**的，双向通信至少需要两个 SA；若同时使用 AH 和 ESP，每种协议都需一对 SA。
- SA 由三元组唯一标识：**安全参数索引 SPI、目的 IP 地址、安全协议（AH 或 ESP）**。

建立 SA 的两种方式：

- **手工方式**：全部参数手工配置，较复杂，但可不依赖 IKE 独立实现。适合对等体数量少或小型静态环境。
- **IKE 动态协商方式**：只需配置 IKE 协商参数，由 IKE 自动创建维护 SA，相对简单。适合中大型动态网络。

## 封装模式

IPSec 有两种封装模式：

### 传输模式

在 IP 报头和高层协议之间插入 AH 或 ESP 头，主要对上层协议数据提供保护。AH 或 ESP 报头位于 IP 报头和传输层报头之间。

- **传输模式 AH**：在 IP 头后插入 AH 头，对整个 IP 数据包做完整性校验。
- **传输模式 ESP**：在 IP 头后插入 ESP 头、数据后插入尾部和认证字段；对高层数据和 ESP 尾部加密，对 ESP 头、高层数据和 ESP 尾部做完整性校验。
- **传输模式 AH+ESP**：综合两者。

### 隧道模式

IPSec 会另外生成一个新的 IP 报头，封装在 AH 或 ESP 之前。隧道模式可以完全对原始 IP 数据报进行认证和加密，还能用对等体 IP 隐藏客户机 IP。

- **隧道模式 AH**：对整个原始 IP 报文提供完整性检查和认证（认证优于 ESP），但不加密，通常和 ESP 联合使用。
- **隧道模式 ESP**：对整个原始 IP 报文和 ESP 尾部加密，对 ESP 头、原始 IP 报文和 ESP 尾部做完整性校验。
- **隧道模式 AH+ESP**：对原始 IP 报文和 ESP 尾部加密，对除新 IP 头之外的整个 IP 数据包做完整性校验。

{{< callout type="info" >}}
两个安全网关之间建立 IPSec 隧道时，建议使用**隧道模式**，以隐藏通信实际使用的源 IP 和目的 IP。
{{< /callout >}}

## 配置步骤

配置 IPSec VPN 的五个步骤：

1. **网络可达**：先确保收发双方网络层可达，使双方只有建立 IPSec 隧道才能进行 IPSec 通信。
2. **定义数据流**：部分流量无需保护，通过 ACL 定义和区分需要进行 IPSec 处理的"兴趣流"。
3. **配置 IPSec 安全提议**：定义安全协议（AH/ESP）、认证算法、加密算法和封装模式。两端对等体必须使用相同的提议。AH 支持 MD5、SHA-1 认证；ESP 支持两种认证（MD5、SHA-1）和三种加密（DES、3DES、AES）。
4. **配置 IPSec 安全策略**：将兴趣流和安全提议绑定。每条策略用唯一名称和序号标识，分手工建立 SA 和 IKE 协商建立 SA 两类。
5. **在接口上应用安全策略**。

## 配置示例（手工方式）

以下为 RTA 侧的配置，对端 RTB 需对称配置。

```text
# 1. 静态路由保证网络可达
[RTA]ip route-static 10.1.2.0 24 20.1.1.2

# 2. 用高级 ACL 定义兴趣流
[RTA]acl number 3001
[RTA-acl-adv-3001]rule 5 permit ip source 10.1.1.0 0.0.0.255 destination 10.1.2.0 0.0.0.255

# 3. 配置 IPSec 提议
[RTA]ipsec proposal tran1
[RTA-ipsec-proposal-tran1]esp authentication-algorithm sha1

# 4. 配置安全策略（手工方式 manual）
[RTA]ipsec policy P1 10 manual
[RTA-ipsec-policy-manual-P1-10]security acl 3001
[RTA-ipsec-policy-manual-P1-10]proposal tran1
[RTA-ipsec-policy-manual-P1-10]tunnel remote 20.1.1.2
[RTA-ipsec-policy-manual-P1-10]tunnel local 20.1.1.1
[RTA-ipsec-policy-manual-P1-10]sa spi outbound esp 54321
[RTA-ipsec-policy-manual-P1-10]sa spi inbound esp 12345
[RTA-ipsec-policy-manual-P1-10]sa string-key outbound esp simple huawei
[RTA-ipsec-policy-manual-P1-10]sa string-key inbound esp simple huawei

# 5. 在接口上应用安全策略
[RTA]interface GigabitEthernet 0/0/1
[RTA-GigabitEthernet0/0/1]ipsec policy P1
[RTA-GigabitEthernet0/0/1]quit
```

{{< callout type="warning" >}}
配置 SPI 时，**本端入方向 SA 的 SPI 必须等于对端出方向 SA 的 SPI**，本端出方向必须等于对端入方向。两端的安全提议参数也必须一致。
{{< /callout >}}

验证命令：

```text
display ipsec proposal      # 查看提议参数（封装模式、安全协议、认证/加密算法）
display ipsec policy        # 查看策略名称、序号、ACL、隧道两端地址、出入方向 SA 参数
```

## 命令详解

| 命令 | 作用 |
| --- | --- |
| `ipsec proposal <name>` | 创建 IPSec 提议并进入提议视图（缺省 ESP + DES + MD5 + 隧道模式） |
| `transform [ah \| ah-esp \| esp]` | 配置隧道采用的安全协议 |
| `encapsulation-mode {transport \| tunnel}` | 配置封装模式 |
| `esp authentication-algorithm [md5 \| sha1 \| sha2-256 \| sha2-384 \| sha2-512]` | 配置 ESP 认证算法 |
| `esp encryption-algorithm [des \| 3des \| aes-128 \| aes-192 \| aes-256]` | 配置 ESP 加密算法 |
| `ah authentication-algorithm [md5 \| sha1 \| ...]` | 配置 AH 认证算法 |
| `ipsec policy <name> <seq> [manual]` | 创建 IPSec 策略并进入策略视图（seq 越小优先级越高，一组最多 16 条） |
| `security acl <acl-number>` | 指定策略引用的 ACL |
| `proposal <name>` | 指定策略引用的提议 |
| `tunnel local {ip \| interface}` / `tunnel remote <ip>` | 配置隧道本端/对端地址 |
| `sa spi {inbound \| outbound} {ah \| esp} <spi>` | 设置 SA 的安全参数索引 |
| `ipsec policy <name>`（接口下） | 在接口上应用安全策略组（手工方式只能应用到一个接口） |
| `display ipsec policy [brief \| name <name> [seq]]` | 查看指定或所有 IPSec 策略 |

## 小结

- **SA 的作用**：定义 IPSec 通信对等体间使用的数据封装模式、认证和加密算法、密钥等参数。
- **兴趣流的处理**：经 IPSec 过滤后的兴趣流，会按 SA 协商的各种参数进行处理并封装，再通过 IPSec 隧道转发。
