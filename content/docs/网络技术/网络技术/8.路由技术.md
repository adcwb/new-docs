路由是把数据包从源送到目的地的过程。这篇讲路由器工作原理、路由的选择规则（最长匹配、优先级、度量值）、静态路由与缺省路由，以及两种典型的动态路由协议 RIP 与 OSPF。

## 路由基础

**路由器用来隔离广播域**，本身负责转发数据，但不转发广播包。路由器可以有多个接口，每个接口是一个广播域；若两个接口配在同一网段会报错。

**什么是路由？** 路由是将数据包从源发送到目的地的过程，路由器可以分割网络。

**路由器工作原理（收到数据如何处理）**：当路由器收到数据包后，查看是不是发给自己的，是就留下；不是就查路由表，路由表中有对应条目就从指定接口转发，没有对应条目则**直接丢弃**。

两类协议要区分清楚：

- **路由协议**：产生并维护路由表信息的协议，为转发提供路径。例如静态路由、RIP、OSPF、IS-IS、BGP。
- **被路由协议（可路由协议）**：定义数据包各字段格式、能被路由表转发的网络层协议，例如 IP。

IP 路由表说明：

- 路由表包含路由器可达的目的网络；目的网络不在表中的数据包会被丢弃。
- 若有多条匹配目的网络的路由，会选择**掩码最长**的那条。
- 默认情况下，路由表只存在直连信息。

按来源，路由表中的路由通常分三类：

1. 链路层协议发现的路由（直连路由）。
2. 网络管理员手工配置的静态路由。
3. 动态路由协议发现的路由。

## 路由选择规则

当路由器有多条候选路由时，依次按以下三条规则决定：

### 1. 最长匹配原则

路由器提取报文的目的 IP，与路由表各表项的掩码做"与"运算，结果与该表项的目的网络比较，相同则匹配。与所有表项匹配后，**选择掩码最长的匹配项**。

### 2. 路由优先级（管理距离）

当多条路由都符合最长匹配时，比较**协议优先级（Preference，即管理距离 Administrative Distance）**。取值范围 0~255，**值越小优先级越高**。不同路由协议来源的路由用它来判断可信度，只把最优的放入路由表。

### 3. 路由度量值（metric）

如果优先级也相同（同一种路由协议学到的），则比较**度量值 metric**，值小者优先。常见度量值有：跳数、带宽、时延、代价、负载、可靠性等。不同协议的度量定义不同：

- RIP：跳数（hop）。
- OSPF：代价（cost）。
- IGRP / EIGRP：复合度量值。

**选择最优路由的顺序**：先比优先级 → 再比 metric → 若仍相同，多条等价路由实现**等价负载分担**。

## 静态路由

静态路由由管理员手动配置和维护。

```text
ip route-static  目标网络  掩码  下一跳地址
```

要点：

- 在串行接口上可指定下一跳地址或出接口；在以太网等广播型接口上配置静态路由**必须指定下一跳地址**。
- 静态路由支持到同一目的地的等价负载分担。

### 缺省路由

**缺省路由**是目的地址和掩码都为全 0（`0.0.0.0/0`）的特殊路由，可匹配任意 IP，一般用在边界路由器上。当报文目的地址无法匹配路由表中任何一项时，路由器按缺省路由转发。

### 浮动静态路由

浮动静态路由在主路由失效时才会加入路由表承担转发：

- 主链路正常时，只有主路由出现在路由表中。
- 主链路故障时，浮动静态路由被激活并加入路由表。

## 动态路由协议分类

**按是否携带掩码（类别）分：**

- **有类路由协议**：更新时不带子网掩码，要求同一网络内掩码连续，在类边界自动汇总。代表：RIPv1、IGRP。
- **无类路由协议**：更新时携带子网掩码，支持 VLSM，汇总可人工控制。代表：RIPv2、EIGRP、OSPF、IS-IS、BGPv4。

**按使用地点分：**

- **IGP（内部网关协议）**：运行于自治系统内部。代表：RIP、IGRP、EIGRP、OSPF。
- **EGP（外部网关协议）**：连接不同自治系统。代表：BGP。

> **自治系统（AS）**：处于一个统一管理机构控制之下的路由器和网络的集合，遵循单一明确的路由策略，分配同一个 AS 编号。一个企业网络通常可视为一个 AS。

**按更新算法分：**

- **距离矢量路由协议**：RIP、IGRP、EIGRP。
- **链路状态路由协议**：OSPF、IS-IS。
- **复合型路由协议**：EIGRP。

## RIP

**RIP（Routing Information Protocol，路由信息协议）**是一种基于**距离矢量**算法的协议，使用**跳数**作为度量。主要应用于规模较小的网络。

特点：配置简单、易于维护、适合小型网络。

工作原理：路由器运行 RIP 后，先发送路由更新请求；收到请求的路由器用自己的 RIP 路由响应。网络稳定后，路由器**周期性**发送路由更新信息。

定时器：更新周期 30 秒、失效时间 180 秒、保持时间 180 秒、刷新时间 240 秒。

{{< callout type="info" >}}
**水平分割**：路由器从某个接口学到的路由，不会再从该接口发回给邻居，用于防止路由环路。
{{< /callout >}}

两个版本对比：

| | RIPv1 | RIPv2 |
| --- | --- | --- |
| 类别 | 有类，不支持 VLSM/CIDR | 无类，支持 VLSM、路由聚合、CIDR |
| 报文发送 | 广播 | 广播或组播（224.0.0.9） |
| 认证 | 不支持 | 支持明文和 MD5 密文认证 |

## OSPF

**OSPF（Open Shortest Path First，开放式最短路径优先）**是 IETF 定义的、基于**链路状态**的内部网关协议（IGP）。RIP 存在收敛慢、易产生环路、可扩展性差等问题，已逐渐被 OSPF 取代。

与距离矢量（"传闻路由"，邻居告诉你就信）不同，链路状态协议让每台路由器都拥有一张完整的"地图"（LSDB），自己用 SPF 算法计算到各目的地的最优路径。

OSPF 特点：无环路、收敛快、扩展性好、支持认证。

### 工作原理（三步）

1. **建邻居**：每台路由器尝试与物理邻居建立邻居关系，放入邻居表（毗邻数据库）。
2. **生拓扑**：向邻居发送链路状态信息，对方放入拓扑表（链路状态数据库 LSDB）。直到学习到网络所有链路状态。
3. **算路由**：每台路由器以自己为中心，用 **SPF 算法（Dijkstra 算法）** 算出到各目的地的最优路径，放入路由表。

三张表：

| 表 | 别名 | 内容 |
| --- | --- | --- |
| Neighbor 表 | 毗邻数据库 | 列出所有识别出的邻居 |
| Topology 表 | 链路状态数据库 LSDB | 一个区域内的路由器和链路，每台路由器都有一份相同的 LSDB |
| Routing 表 | 转发数据库 | 到各目的地的最优路径 |

几个术语：

- **链路（link）**：在 OSPF 中，链路与接口含义相同。
- **邻居（neighbor）**：建立了连接的相邻 OSPF 路由器，且接口在同一区域，通过 Hello 包发现。
- **邻接（adjacency）**：为交换路由信息在邻居间建立的逻辑连接，已经相互传递了 LSA。只是邻居而没继续传 LSA 的，就只是邻居关系而非邻接关系。

### 五种报文

OSPF 报文封装在 IP 中，协议号 **89**。从邻居到邻接的过程中会用到五种报文：

| 报文 | 作用 |
| --- | --- |
| Hello | 发现邻居、建立维护邻接关系。目标 IP `224.0.0.5`，每 10 秒发一个，默认 40 秒收不到就认为邻居失效 |
| DD（Database Description） | 链路状态摘要，类似目录 |
| LSR（LSA Request） | 链路状态请求，请求缺少的链路信息 |
| LSU（LSA Update） | 链路状态更新，回复真正的链路信息 |
| LSAck（Link State Ack） | 链路状态确认 |

### 邻居到邻接的状态机

1. **Down**：刚配好 OSPF，还没发信息。
2. **Init**：开始发送 Hello 包。
3. **Two-way**：收到的对方报文中含有自己的 Router ID，双向通信建立。
4. **Exstart**：协商主从关系，决定谁先发 DD。
5. **Exchange**：双方交换 DD 报文（目录摘要）。
6. **Loading**：互相请求缺少的链路状态（LSR/LSU/LSAck）。
7. **Full**：双方 LSDB 同步完成，正式成为邻接关系。

验证命令：`display ospf peer brief`。

### 网络类型与 DR/BDR

缺省情况下，OSPF 认为以太网是**广播类型**，PPP、HDLC 是**点到点类型**。查看命令：`display ospf interface 接口`。

- **点到点**：不选举 DR/BDR。
- **广播类型（MA）**：选举 **DR 和 BDR**。

为什么要选 DR/BDR？非 DR/BDR 的路由器称 DRother，只与 DR/BDR 建立邻接、彼此间只是邻居，从而**减少 LSA 泛洪带来的开销**。

DR/BDR 选举依据：

1. **优先级**：Hello 包中携带，默认 1，范围 0~255，**大者优先**（优先级 0 不参与选举）。
2. **Router ID**：优先级相同时，Router ID 大者当选。

### Router ID

- 特点：IP 格式的 ID，但并非真实 IP；唯一且任意。
- 自动选择：优先选**环回口**地址（环回口永不 down），否则选最大的物理口 IP。
- 手动指定：`ospf` 进程下 `router-id x.x.x.x`。

### 组播地址

- `224.0.0.5`：建立邻接、发送 Hello 时使用，代表所有 OSPF 路由器。
- `224.0.0.6`：只有 DR/BDR 处理。网络拓扑变化时，其他路由器把更新发给 DR/BDR（用 224.0.0.6）；DR/BDR 再通知其他路由器时用 224.0.0.5。

### OSPF 区域

- **区域 0**：骨干区域。
- **区域 1、2、3……**：非骨干区域。
- **ABR（区域边界路由器）**：连接骨干区域和非骨干区域。
- **ASBR（自治系统边界路由器）**：运行非 OSPF 协议，通过某些方法把外部路由引入 OSPF。

### 基本配置

```text
[RTA]router id 1.1.1.1                              # 手动指定路由器 ID
[RTA]ospf                                            # 进入 OSPF 进程
[RTA-ospf-1]area 0                                   # 进入区域
[RTA-ospf-1-area-0.0.0.0]network 10.1.1.0 0.0.0.255  # 宣告直连网段（反掩码）
```

常用查看命令：

```text
display ospf peer brief                  # 查看邻居、Router ID
display ospf interface                   # 查看接口开销、网络类型
display ip routing-table protocol ospf   # 查看 OSPF 路由表
reset ospf process                       # 重置 OSPF 进程
```

## 路由引入（重发布）

当网络中存在多种路由协议（静态、缺省、RIPv1/v2、OSPF）时，如何实现全网互通？例如总公司收购子公司，两者使用的协议不同。

- 华为称**路由引入**，思科称**路由重发布**。
- 做法：在核心（ASBR）路由器上同时运行多种协议，对不同协议的区域实现转换。

常用命令：

```text
import-route static/ospf/rip cost <开销>   # 把其它协议路由引入本进程（华为）
default-route-advertise                    # OSPF 缺省路由宣告
default-route originate                     # RIP 缺省路由起源
```

## 二层/三层接口切换（华为 AR 补充）

```text
portswitch        # 将以太网接口从三层模式切换到二层模式
undo portswitch   # 从二层模式切换到三层模式
```

缺省情况下，以太网接口工作在二层模式。不同型号（AR150-S/200-S、AR121-S、AR2201、AR160-S 等）对接口切换的支持范围不同，配置前需查阅对应型号说明。
