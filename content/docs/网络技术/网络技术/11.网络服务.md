这篇讲两个最常用的网络服务：**DHCP**（自动分配 IP 地址）和 **NAT**（私网与公网地址转换）。它们分别解决了"如何让主机自动获取地址"和"如何让大量私网主机共享公网地址访问外网"两个核心问题。

## DHCP

**DHCP（Dynamic Host Configuration Protocol，动态主机配置协议）**用于自动给主机分配 IP 地址、网关、DNS 等配置。

### 报文类型

| 报文类型 | 含义 |
| --- | --- |
| DHCP DISCOVER | 客户端用来寻找 DHCP 服务器 |
| DHCP OFFER | 服务器响应 DISCOVER，携带各种配置信息 |
| DHCP REQUEST | 客户端请求配置确认，或续借租期 |
| DHCP ACK | 服务器对 REQUEST 的确认响应 |
| DHCP NAK | 服务器对 REQUEST 的拒绝响应 |
| DHCP RELEASE | 客户端释放地址时通知服务器 |

### 工作过程

DHCP 获取地址的四步交互：

1. **DHCP Discover**（广播）：客户端寻找服务器。
2. **DHCP Offer**（单播）：服务器提供可用地址及配置。
3. **DHCP Request**（广播）：客户端请求使用某个地址。
4. **DHCP ACK**（单播）：服务器确认，分配完成。

### 租约更新

- IP 租约期限到达 **50%** 时，客户端请求更新租约。
- 到达 **87.5%** 仍未收到响应，客户端申请**重绑定**（向其他 DHCP 服务器请求）。
- 租约到期前都没收到响应，客户端**停止使用**该 IP。
- 客户端不再使用地址时，也可主动发送 **DHCP RELEASE** 释放该 IP。

> 华为 ARG3 系列路由器支持两种地址池：**全局地址池**和**接口地址池**。

## NAT

随着 Internet 发展，IPv4 地址枯竭成为瓶颈。在 IPv6 广泛应用之前，**NAT** 是缓解这一问题的主要手段。

**NAT（Network Address Translation，网络地址转换）**主要用于让内部网络的主机访问外部网络。当局域网内主机访问外网时，NAT 把其**私网地址转换为公网地址**，并且多个私网用户可以共用一个公网地址，既保证互通又节省公网地址。

基本概念：

- 企业/家庭网络是私有网络，使用私有地址；运营商网络是公共网络，使用公有地址。**私有地址不能在公网中路由**。
- NAT 一般部署在连接内网和外网的网关设备上。

### NAT 的几种类型

| 类型 | 说明 |
| --- | --- |
| 静态 NAT | 实现私有地址和公有地址的**一对一**映射，一个公网 IP 固定分配给唯一内网主机 |
| 动态 NAT | 基于**地址池**实现私有与公有地址的转换 |
| NAPT | 网络地址端口转换，允许多个内部地址映射到同一公有地址的**不同端口** |
| Easy IP | 允许多个内部地址映射到**网关出接口地址**的不同端口 |
| NAT Server | 通过配置 NAT 服务器，可使外网用户访问内网服务器 |
