这篇讲网络分层的思想：为什么要分层、OSI 七层参考模型各层的职责，以及数据在层间封装与解封装的过程。网络模型是整个网络知识体系的"地图"。

## 为什么要分层

网络通信非常复杂。分层的本质是**把网络任务分解到不同的层**，每层只关注自己的职责，从而降低复杂性。

分层设计的意义：

- **功能独立**：每层完成特定功能，互不干扰。
- **模块化生产**：厂商可以只实现某几层，产品之间通过标准接口协作。
- **便于学习与排错**：可以聚焦讨论某一层的协议细节，定位故障也更容易。

> 分层的核心规则：**每层利用紧邻下层提供的服务，并向紧邻上层提供服务。**

## 什么是协议

为了让数据能在网络上从源传递到目的地，网络中所有设备需要"讲"相同的"语言"。**描述网络通信中这套"语言"规范的一组规则，就是协议。**

## 网络模型的分类

网络模型大体分为两类：

- **网络设计模型**：思科（Cisco）设计大型网络时提出的分层思想，用于指导组网，提供多种应用解决方案。
- **网络通信模型**：描述数据如何通信的参考模型，包括 **OSI/RM 模型**和 **TCP/IP 协议栈**。

## 网络设计分层模型

这是面向"如何组网"的层次模型，把网络设备按职责分为三层：

| 层次 | 功能 | 典型设备 |
| --- | --- | --- |
| 核心层 | 实现数据的快速转发（一般不做路由选择、ACL、包过滤） | 核心交换机 |
| 汇聚层（分布层） | 路由控制、安全、介质转换、远程接入 | 路由器、多层交换机 |
| 接入层 | 提供大量接口，把终端接入网络，提供本地服务 | 交换机、集线器、AP（Access Point） |

## OSI 参考模型

OSI（Open System Interconnection，开放式系统互联）由 ISO 制定，将网络通信划分为七层。

| 层 | 作用 | 数据单位（PDU） | 典型协议 |
| --- | --- | --- | --- |
| 应用层 | 用户或程序接口 | data | Telnet、FTP、HTTP |
| 表示层 | 数据的表现形式、加密、压缩 | data | ASCII、JPEG |
| 会话层 | 应用会话的建立、管理与同步 | data | — |
| 传输层 | 可靠/不可靠传输、错误检测、流控，定义端到端传递方式 | 段（segment） | TCP、UDP |
| 网络层 | 提供逻辑（IP）地址、选路，定义端到端传递路径 | 包（packet） | IP、ARP、IPX |
| 数据链路层 | 成帧、用 MAC 地址访问介质、差错检测 | 帧（frame） | Ethernet、PPP |
| 物理层 | 设备间比特流传输、物理接口、电气特性 | 比特（bit） | IEEE 802.3、V.35 |

注：**下层为上层提供服务，同一层次之间使用相同的协议。**

按职能可以把七层再归为两组：

- **面向应用的上层**：应用层、表示层、会话层。
- **面向数据流的下层**：传输层、网络层、数据链路层、物理层。

## 封装与解封装

数据在不同层有不同的称呼（PDU，协议数据单元）：

- 传输层：段（segment）
- 网络层：包（packet）
- 数据链路层：帧（frame）
- 物理层：位（bit）

**封装**：发送数据时，数据流从上至下传输。每经过一层都会在数据前后添加该层的头部（部分层还有尾部）信息，便于接收方处理。

**解封装**：接收数据时，数据流从下至上传输，每层去掉本层的头部和尾部后，把数据交给上一层。

层间通信分两种场景：

- **计算机内部通信**：相邻上下层之间通过标准接口交互。
- **计算机之间通信**：发送方与接收方的**对等层**之间使用相同协议进行通信。

{{< callout type="info" >}}
数据链路层的帧尾部包含 **FCS（帧校验序列，Frame Check Sequence）**，用于接收方检测帧在传输中是否出错。
{{< /callout >}}

## OSI 与 TCP/IP 的对应

实际工程中广泛使用的是 TCP/IP 模型，它把 OSI 的上三层合并为应用层、下两层合并为网络接口层：

| TCP/IP 模型 | 对应 OSI 层 |
| --- | --- |
| 应用层 | 应用层 + 表示层 + 会话层 |
| 传输层 | 传输层 |
| 网络层 | 网络层 |
| 网络接口层 | 数据链路层 + 物理层 |

应用层的封装由应用程序完成，用户无需关心；下面的传输层、网络层、网络接口层是网络工程师重点学习的对象。
