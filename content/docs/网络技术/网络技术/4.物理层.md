物理层负责在设备之间传输**比特流**，定义介质类型、连接器、信号特性。这篇讲传输介质（光纤、双绞线、同轴电缆等）、双绞线线序标准、光纤接口类型、线缆命名规范，以及集线器和 CSMA/CD 冲突避免机制。

## 物理层职责

物理层定义：介质类型、连接器类型、信号类型。

常见协议规范：Ethernet、IEEE 802.3、EIA/TIA-232、V.35。其中以太网和 802.3 用于定义 LAN，EIA/TIA-232 和 V.35 用于定义 WAN。

工作在物理层的设备：传输介质、Hub（集线器）。处理的数据类型是 **bit**。

## 传输介质

传输介质分为无线和有线两大类：

- **无线介质**：微波、红外线、蓝牙、卫星、射频识别（RFID）、NFC 等。
- **有线介质**：光纤、双绞线、同轴电缆（有线电视线缆）、电话线、串行线等。

### 光纤

传输距离从几百米到几十公里，带宽有 100M、1000M、10G。

| 类型 | 特点 | 传输距离 |
| --- | --- | --- |
| 单模光纤 | 传输单一模式的光，频带宽、传输距离远 | 1G 带宽 > 5000m |
| 多模光纤 | 传输多种模式的光，距离相对较近 | 10~100M 带宽 < 2000m；1G 带宽 < 550m |

### 双绞线

传输距离在 100 米以内，带宽有 10M、100M、1000M。

按抗干扰能力分为：

- **STP**（屏蔽双绞线）
- **UTP**（非屏蔽双绞线）

按制线线序分为：

- **直通线**（顺序线）：用于**异种设备**的连接。
- **交叉线**：用于**同种设备**的连接。
- **反转线**：用于终端设备控制（Console）的连接。

{{< callout type="info" >}}
**如何区分"同种设备"和"异种设备"？**

- 工作在**网络层及以上**的设备为一类，如路由器、PC。
- 工作在**数据链路层及以下**的设备为一类，如交换机（二层）、集线器。

例如：路由器与 PC 相连用交叉线（同类）；交换机与 PC 相连用直通线（异类）。
{{< /callout >}}

双绞线线序有两个标准：

| 标准 | 线序 |
| --- | --- |
| T568A | 绿白、绿、橙白、蓝、蓝白、橙、棕白、棕 |
| T568B | 橙白、橙、绿白、蓝、蓝白、绿、棕白、棕 |

- 直通线：两端都用 T568B（或都用 T568A）。
- 交叉线：一端 T568A，另一端 T568B。

### 同轴电缆

带宽 10M：

- 粗同轴电缆：传输距离约 500m。
- 细同轴电缆：传输距离约 200m（185m）。

### 串行线

用于广域网连接，区分两端角色：

- **DCE**（数据通信设备）：需要提供时钟频率。
- **DTE**（数据终端设备）：使用 DCE 提供的时钟。

## 光纤接口

光纤接口用以稳定（但非永久）地连接两根或多根光纤的无源组件：

| 接口 | 说明 |
| --- | --- |
| FC | 圆形带螺纹光纤接头 |
| ST | 卡接式圆形光纤接头（最常见） |
| SC | 方型光纤接头 |
| LC | 窄体方形光纤接头 |
| MT-RJ | 收发一体 |

## 线缆命名规范

线缆命名形如 `10base2`、`100baseTX`：

| 命名 | 含义 |
| --- | --- |
| 10base2 | 10M 细同轴电缆（约 200m） |
| 10base5 | 10M 粗同轴电缆（约 500m） |
| 10baseT | 10M 双绞线 |
| 100baseTX | 100M 双绞线 |
| 100baseFX | 100M 光纤 |

命名规则：`10/100/1000` 表示带宽；`base` 表示基带传输（数字信号）；末尾的 `2/5/T/F` 分别表示介质类型——细缆 / 粗缆 / 双绞线 / 光纤。

补充概念：

- **基带**：传输数字信号。
- **宽带**：传输模拟信号。
- **RJ-45**：水晶头，网络接口。
- **RJ-11**：电话接口。

## 集线器与冲突避免

集线器基于星型拓扑，功能是连接终端设备、放大并传输信号。

工作原理：将收到的比特位进行校正放大后**复制并从所有其它接口泛洪**出去。

集线器特性总结：

1. 工作在物理层，处理 BIT。
2. 对信号进行校正、放大、泛洪。
3. 所有设备处于**同一个冲突域**、**同一个广播域**，并**共享带宽**。
4. 采用 CSMA/CD 机制减少冲突。
5. 网络带宽利用率仅 30%~40%。

### CSMA/CD

**CSMA/CD**（Carrier Sense Multiple Access with Collision Detection，带冲突检测的载波侦听多路访问）用于解决共享介质上的数据冲突：

- **CS**（载波侦听）：发送前先监听信道是否空闲。
- **MA**（多路访问）：多个节点共享同一信道。
- **CD**（冲突检测）：边发送边监听，发现冲突立即停止。

工作流程：发送数据前先检测网络是否空闲；若不空闲就继续检测，空闲则发送。发送时边发边听，如果发生碰撞，在未超过 16 次碰撞的情况下，等待一个随机时间（退避，一般 96bit）后再尝试；若超过 16 次则放弃发送。

## 接口通信模式

| 模式 | 类比 | 说明 |
| --- | --- | --- |
| 单工 | 电台 | 只允许甲方向乙方传送，乙方不能反向传送 |
| 半双工 | 对讲机 | 数据可双向传输，但不能同时进行（集线器同一时刻只能单向传输） |
| 全双工 | 手机 | 数据可在两个方向上同时传输，相当于两个单工的结合 |
