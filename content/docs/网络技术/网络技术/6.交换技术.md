这篇讲交换网络中的进阶技术：用 STP/RSTP 防止二层环路、用 VLAN 隔离广播域、用链路聚合提升带宽与冗余，以及 GARP/GVRP 与 WLAN 的基本概念。

## STP 生成树协议

交换网络中如果存在冗余链路，二层环路会引发广播风暴。**STP（Spanning Tree Protocol，生成树协议）**通过阻塞部分端口，把有环的物理拓扑裁剪成无环的逻辑树。

### 端口角色

| 端口角色 | 描述 |
| --- | --- |
| 根端口（Root Port） | 所在交换机上离根桥最近的端口，处于转发状态 |
| 指定端口（Designated Port） | 负责转发所连网段去往/来自根桥方向的数据 |
| 预备端口（Alternate Port） | 阻塞作为备用，不转发任何数据 |

几个关键标识：

- **网桥 ID（BID）**：交换机的身份标识，全局唯一（由优先级 + MAC 地址组成）。
- **端口 ID（PID）**：交换机端口的身份标识（由端口优先级 + 端口编号组成）。
- **根路径成本**：各网桥去往根桥的开销，由沿途各段路径成本（Path Cost）叠加而来。

### STP 工作原理（四步）

1. **选举根网桥（Root Bridge）**：比较 BID，先比优先级（小者优先），相同则比 MAC 地址（小者为根）。
2. **选举根端口（Root Port）**：每台非根交换机选到根桥代价最小的端口；相同则比上游网桥 BID、再比上游端口 PID，值小者优先。
3. **选举指定端口（Designated Port）**：选举过程同根端口。**根桥上的所有端口都是指定端口**；与根端口直连的对端端口也是指定端口；其余按路径开销、BID、PID 判定。
4. **阻塞预备端口（Alternate Port）**：未被选为根端口或指定端口的端口被阻塞。

### 端口状态

| 端口状态 | 转发数据帧 | 学习 MAC | 处理 BPDU | 说明 |
| --- | --- | --- | --- | --- |
| Disabled | 否 | 否 | 否 | 端口未启用 |
| Blocking | 否 | 否 | 只收不发 | 阻塞状态 |
| Listening | 否 | 否 | 收发 | 侦听，参与生成树计算 |
| Learning | 否 | 是 | 收发 | 学习状态 |
| Forwarding | 是 | 是 | 收发 | 正常转发 |

{{< callout type="info" >}}
**RSTP（Rapid STP，快速生成树）**是 STP 的改进版，通过简化端口状态、引入提议/同意机制，把收敛时间从 STP 的几十秒缩短到秒级。
{{< /callout >}}

## VLAN 虚拟局域网

**VLAN（Virtual Local Area Network，虚拟局域网）**是一组逻辑上的设备和用户，它们不受物理位置限制，可以按功能、部门、应用等因素组织起来，相互通信就像在同一网段中一样。

要点：

- VLAN 工作在 OSI 的第 2 层和第 3 层，**一个 VLAN 就是一个广播域**。
- 默认情况下不同 VLAN（广播域）相互隔离，**VLAN 之间的通信需要通过第 3 层的路由器**完成。

VLAN 的优点：

- 减少网络设备移动、添加和修改的管理开销。
- 控制广播活动，抑制广播风暴。
- 提高网络的安全性（不同部门逻辑隔离）。

## 链路聚合

**链路聚合（Link Aggregation）**是指将多个物理端口捆绑成一个逻辑端口，实现出/入流量在各成员端口间的负荷分担。

工作机制：

- 交换机根据配置的负荷分担策略，决定报文从哪个成员端口发往对端。
- 当检测到某成员端口链路故障时，停止在该端口发送，并在剩余链路中重新计算转发端口；故障端口恢复后再次重新计算。

价值：链路聚合在**增加链路带宽、实现传输弹性与冗余**方面是一项重要技术。如果聚合的每条链路走不同物理路径，还能同时提供冗余与容错，常用于在吉比特以太网交换机之间构建多吉比特主干链路。

## GARP 与 GVRP

**GARP（Generic Attribute Registration Protocol，通用属性注册协议）**定义了一套结构、操作规则和状态机，用于在桥接局域网中声明、注册或注销属性值，使属性信息在整个网络中传播，形成一棵"可达性"树。它是对 IEEE 802.1D（生成树）的 802.1P 扩展的一部分。

GARP 主要包含：

- **GID（GARP 信息声明）**：GARP 生成数据部分。
- **GIP（GARP 信息传播）**：GARP 数据分配部分。
- **GMRP（GARP 组播注册协议）**：为参与者动态注册/注销 MAC 桥信息。

**GVRP（GARP VLAN Registration Protocol）**是 GARP 的一种应用，用于维护 VLAN 的动态注册信息并传播到其他交换机，作用类似 Cisco 的 VTP 协议，同为二层协议。

GVRP 要点：

- 定义在 IEEE 802.1P 标准中，允许对 802.1Q VLAN 进行控制。
- 只运行在 802.1Q 干线（Trunk）链路上，通过裁剪干线使得只有活动 VLAN 才在干线上传输。
- 支持设备之间 VLAN 信息的自动传播：手工配置一台交换机的 VLAN 信息后，网络中其它交换机能动态学习这些 VLAN。
- 在为干线添加某 VLAN 前，GVRP 需先收到来自交换机的 join 信息。

## WLAN 无线局域网

**WLAN（Wireless Local Area Networks，无线局域网）**利用射频（RF）技术、使用电磁波，取代传统双绞铜线在空中进行通信连接，实现"信息随身化、便利走天下"。

基于 IEEE 802.11 标准的 WLAN 允许在 **2.4GHz 或 5GHz** 免授权 ISM 频段进行无线连接，被广泛应用于家庭、企业到互联网接入热点。

典型场景——家庭无线 LAN：一台无线路由器通常集**防火墙、路由器、交换机、无线接入点（AP）** 于一体，可以：

- 保护家庭网络免受外界入侵。
- 共享一个 ISP 分配的公网 IP（配合 NAT）。
- 提供有线以太网口，并通过无线 AP 为多台无线设备服务。

进阶概念：

- 基础模块通常提供 2.4GHz 的 802.11b/g（Wi-Fi）。
- 双波段 AP 同时提供 2.4GHz（802.11b/g/n）和 5GHz（802.11a/n），本质上是两个 AP 合体，可同时提供两个互不干扰的频率。
- MIMO 设备使用多个射频以提升速率。2.4GHz 频段常拥挤，5GHz 相对空闲。
